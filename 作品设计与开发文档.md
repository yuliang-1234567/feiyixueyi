## 第一章 需求分析

### 1.1 开发背景与目标

非物质文化遗产（非遗）是我国优秀传统文化的重要组成部分，但在数字化时代面临传承难、传播窄、应用少等问题。本作品旨在构建一个融合 AI 技术的非遗技艺学习与文创平台，实现非遗的数字化保护、趣味化学习与商业化转化。

**开发动机**：传统非遗学习依赖线下师徒传承，门槛高、周期长；文创设计往往脱离非遗元素，缺乏文化内涵。本平台通过 AI 学艺、数字焕新等功能，降低学习门槛，激发用户对非遗的兴趣，推动非遗与现代生活相结合。

**对标作品**：市面上的非遗类应用多以展示、资讯为主，缺乏互动学习与个性化建议；文创平台多聚焦成品销售，少有「用户创作+AI辅助」的一体化体验。本作品对标「故宫文创」「中国非遗网」等，在功能深度与 AI 融合度上有所创新。

### 1.2 目标用户

- **学生群体**：对非遗感兴趣，希望系统化学习剪纸、书法等技艺
- **文创爱好者**：希望将非遗纹样应用到 T 恤、手机壳等产品设计中
- **文化机构**：需要面向公众的非遗展示与传播平台

### 1.3 主要功能

| 功能模块 | 描述 |
|----------|------|
| AI 学艺 | 用户上传剪纸/书法作品，与参考作品进行轮廓比对，输出相似度评分及个性化学习建议 |
| 数字焕新 | 支持纹样图、文本描述或二者组合，调用火山引擎即梦 AI 生成文创产品效果图 |
| 地域非遗地图 | 基于中国省级行政区划的互动地图，展示各省代表性非遗项目，支持点击探索 |
| 作品展示 | 用户浏览、发布作品，支持点赞、评论与分类筛选 |
| 文创商城 | 商品浏览、加购、下单、订单管理，支持数字焕新生成作品一键发布为商品 |

### 1.4 主要性能要求

- 图片上传：单文件最大 50MB，支持 jpeg/png/gif/webp
- 数字焕新限流：每用户每分钟最多 10 次，避免滥用
- 响应时间：普通页面 < 2s，AI 生成接口 < 90s

### 1.5 竞品分析

| 维度 | 本作品 | 故宫文创 App | 中国非遗网 |
|------|--------|--------------|------------|
| 学习模式 | AI 轮廓比对+智能建议 | 视频、图文为主 | 图文、名录为主 |
| 创作工具 | 纹样+描述生成产品图 | 无 | 无 |
| 地图探索 | 省份互动地图 | 无 | 无 |
| 电商闭环 | 创作→发布→购买 | 成品销售 | 无 |
|          |                      |                |                |

## 第三章 详细设计

### 3.1 界面设计

**典型使用流程 1：AI 学艺**

1. 进入「AI 学艺」页面，选择技能类型（如剪纸）
2. 上传本人创作的剪纸作品图片
3. 系统选择参考作品并进行轮廓比对
4. 展示相似度评分、改进建议及学习计划
5. 可选保存分析结果至个人作品

**典型使用流程 2：数字焕新**

1. 进入「数字焕新」页面
2. 选择：① 上传纹样图 ② 输入文本描述 ③ 纹样+描述
3. 选择产品类型（T 恤、手机壳、帆布袋等）
4. 点击「生成预览」，等待即梦 AI 生成
5. 预览满意后可保存为作品或发布为商品

**典型使用流程 3：地域探索**

1. 首页「地域非遗地图」展示中国省级行政区划
2. 点击某省份，展示该省代表性非遗项目及简介
3. 可进一步跳转作品/商品相关页面

### 3.2 数据库设计

**主要表结构**

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| users | 用户 | id, username, email, password, role, nickname, bio |
| artworks | 作品 | id, title, imageUrl, category, authorId, aiSimilarity, tags |
| artwork_likes | 作品点赞 | userId, artworkId |
| products | 商品 | id, name, images, price, category, patternId, status |
| orders | 订单 | id, userId, items, totalAmount, shippingAddress, status |
| ar_contents | AR 内容 | id, markerId, model3d, animation, category |
| masters | 非遗大师 | id, name, category, description, bio |
| news | 新闻 | id, title, content, category, publishDate |

**ER 关系简述**

- users 1:N artworks（用户发布多件作品）
- users 1:N orders（用户多个订单）
- artworks 1:N products（作品可关联多商品，patternId 指向作品）
- users N:N artworks（通过 artwork_likes 实现点赞）

**设计说明**：orders 的 items 使用 JSON 存储订单明细，便于记录商品快照与数量，满足订单历史查询需求，采用适度反范式以简化查询。

### 3.3 关键算法与技术

**（1）轮廓相似度算法（AI 学艺）**

将用户作品与参考作品统一缩放至 224×224 像素并转为灰度图，基于像素级差异计算结构相似度：

```
1. 使用 Sharp 提取两图 raw 像素数据
2. 计算像素差：diff_i = |pixels1[i] - pixels2[i]|
3. 平均差异：meanDiff = Σdiff / pixelCount
4. 归一化：normalizedDiff = meanDiff / 255
5. 相似度：similarity = exp(-normalizedDiff × 3)
6. 输出 0–100 的评分，并结合 DeepSeek 生成学习建议
```

**（2）数字焕新流程**

- 输入校验：纹样或描述至少一种；描述 ≤ 800 字；文件 ≤ 50MB
- 限流：每用户每分钟最多 10 次
- 调用逻辑：有纹样 → 火山引擎图生图；仅描述 → 文生图；纹样+描述 → 图生图
- 使用 @volcengine/openapi 进行 AK/SK 签名，请求超时 60s，失败重试 2 次

**（3）地域非遗地图**

- 使用 ECharts 注册中国省级 GeoJSON（来源：阿里云 DataV）渲染地图
- 省份点击触发回调，根据省份名称匹配 provincesData 中的非遗内容并展示

## 第五章 安装及使用

### 5.1 环境要求

- Node.js >= 16
- MySQL >= 8.0
- npm 或 yarn

### 5.2 安装步骤

**1. 克隆项目并安装依赖**

```
# 进入项目目录
cd ihc

# 安装后端依赖
cd backend && npm install

# 安装前端依赖
cd ../web && npm install
```

**2. 配置数据库**

```
# 创建数据库并导入
mysql -u root -p < backend/database/init.sql
```

**3. 配置环境变量**

在 `backend` 目录创建 `.env` 文件：

```
PORT=3000
NODE_ENV=development
DB_HOST=localhost
DB_PORT=3306
DB_NAME=ihc
DB_USER=root
DB_PASSWORD=你的数据库密码
JWT_SECRET=随机密钥
JWT_EXPIRE=7d
DEEPSEEK_API_KEY=你的DeepSeek密钥
VOLCENGINE_ACCESS_KEY=火山引擎AK
VOLCENGINE_SECRET_KEY=火山引擎SK
VOLCENGINE_REGION=cn-north-1
VOLCENGINE_JIMENG_ENDPOINT=https://visual.volcengineapi.com
CORS_ORIGIN=http://localhost:3001
```

在 `web` 目录创建 `.env`：

```
REACT_APP_API_URL=http://localhost:3000/api
```

**4. 启动服务**

```
# 启动后端
cd backend && npm run dev

# 启动前端（新终端）
cd web && npm start
```

后端默认运行于 http://localhost:3000，前端于 http://localhost:3001。

### 5.3 默认账号

- 测试账号：user1 / admin123
- 管理员：admin / admin123

### 5.4 典型使用流程

1. 访问 http://localhost:3001，点击「注册」创建账号
2. 登录后进入「AI 学艺」，上传剪纸或书法作品，查看相似度与建议
3. 进入「数字焕新」，输入描述或上传纹样，选择产品类型，生成预览
4. 在「作品」中浏览他人作品，在「商城」中浏览与购买商品

## 第六章 项目总结

### 6.1 开发体会

在开发过程中，团队对非遗数字化、AI 赋能文化传承有了更深理解。技术方面，前后端分离与模块化设计提升了协作效率；与 DeepSeek、火山引擎等外部 API 的对接，锻炼了接口集成与异常处理能力。产品方面，从「展示」到「学习+创作+消费」的闭环设计，使平台更具实用价值。

### 6.2 困难与解决

- **跨平台兼容**：初期 TensorFlow.js Node 在 Windows 上编译困难，改为 Sharp+ 外部 API 方案，兼顾稳定与功能
- **API 签名**：火山引擎即梦需 Volc-Signature-V4 签名，通过引入 @volcengine/openapi 官方 SDK 解决

### 6.3 后续升级方向

- 引入更精细的轮廓/风格相似度算法，提升 AI 学艺准确度
- 地图数据接入具备审图号的合规地图服务
- 小程序端完善 AR 识物与支付流程
- 探索非遗知识图谱与推荐算法

### 6.4 商业推广设想

可与非遗保护机构、文创企业合作，提供 B 端定制服务；将数字焕新能力开放为 API，供第三方平台调用；开展非遗进校园活动，扩大用户基础。
