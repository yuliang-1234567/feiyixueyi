## 本地环境保姆级搭建与启动指南（仅前端 + 后端 + 数据库）

> 本文面向**零基础 / 少配置经验**的同学，从「什么都没有」开始，手把手完成本项目的**后端服务 + MySQL 数据库 + Web 前端**的本地搭建和启动。  
> **不包含微信小程序相关内容**。

---

### 一、整体架构一眼看懂

- **后端（backend）**
  - 技术栈：Node.js + Express + Sequelize + MySQL
  - 主要功能：用户、作品、产品、订单、AI 学艺、数字焕新、AR 等接口
  - 监听端口：默认 `3000`
- **数据库（MySQL）**
  - 使用 Sequelize 连接 MySQL
  - 默认配置：
    - 数据库名：`ihc`
    - 用户名：`root`
    - 密码：`123456`
    - 主机：`localhost`
    - 端口：`3306`
- **Web 前端（web）**
  - 技术栈：React 18 + Ant Design + axios 等
  - 启动端口：默认 `3000`，本项目通过 `proxy` 或环境变量把请求转发到后端

你最终要实现的是：

- 打开浏览器访问 `http://localhost:3000`（或 React 实际提示的端口），可以正常进入 Web 页面
- 页面上的接口请求都能成功访问到本机的 Node.js 后端和 MySQL 数据库

### 二、准备工作（只要按顺序做就行）

#### 1. 安装 Node.js（含 npm）

1. 访问 Node.js 官网（用浏览器打开）：`https://nodejs.org/`
2. 下载 **LTS 版本**（长期支持版，稳定）
3. 双击安装包，一路「下一步」，保持默认选项即可
4. 安装完成后，打开 PowerShell（或 CMD），输入：

```bash
node -v
npm -v
```

如果能看到版本号（如 `v18.x.x`、`8.x.x`），说明安装成功。

> **建议版本**：Node.js ≥ 16，npm ≥ 8。

#### 2. 安装 Git

1. 访问 Git 官网：`https://git-scm.com/`
2. 下载 Windows 版本安装包
3. 安装过程中保持默认选项即可
4. 安装完成后，打开 PowerShell，输入：

```bash
git --version
```

若能看到版本号（如 `git version 2.x.x`），表示 Git 安装成功。

#### 3. 安装 MySQL 8.0

1. 访问 MySQL 官网上的下载页面：`https://dev.mysql.com/downloads/`
2. 选择 **MySQL Community Server 8.x**，下载 Windows 版本安装程序
3. 安装时注意：
   - 记住设置的 **root 用户密码**，建议先按项目默认使用：`123456`
   - 安装完成后，确保 MySQL 服务会自动启动（一般安装向导会配置好）
4. 验证 MySQL 是否启动：
   - 在 PowerShell 中输入：

```bash
mysql -u root -p
```

输入安装时设置的 root 密码，如果能进入 MySQL 命令行（提示符变为 `mysql>`），说明 MySQL 运行正常。

> 如果你不熟悉 MySQL，也没关系，本项目已经提供了初始化脚本和 Node.js 脚本，按后面步骤照做即可。

#### 4. 建议安装的可视化工具（可选）

- **VS Code / Cursor**：方便编辑代码和查看日志
- **MySQL Workbench**：可视化管理 MySQL（不想用命令行时用它）

### 四、配置环境变量（.env）

本步骤非常关键，关系到端口、数据库连接、跨域等。

#### 1. 后端 `.env` 配置（必做）

在 `backend` 目录下创建一个名为 `.env` 的文件（**无后缀名**），内容可以从 `backend/ENV_CONFIG.md` 复制一份基础模板，然后根据需要调整。  
最基础、能本地跑起来的示例：

```env
# 服务器配置
PORT=3000
NODE_ENV=development

# 数据库配置（MySQL）
DB_HOST=localhost
DB_PORT=3306
DB_NAME=ihc
DB_USER=root
DB_PASSWORD=123456

# JWT 配置
JWT_SECRET=your_jwt_secret_key_here_change_in_production
JWT_EXPIRE=7d

# CORS 配置（前端地址）
CORS_ORIGIN=http://localhost:3000
```

说明：

- `DB_*`：如果你安装 MySQL 时使用的是别的用户/密码，请相应修改这里
- `CORS_ORIGIN`：本地开发时前端一般也是 `http://localhost:3000`，保持一致即可
- 其他如 `OPENAI_API_KEY`、`DEEPSEEK_API_KEY`、`VOLCENGINE_*`：**不填也可以**，只是相关 AI 功能会受限或回退为静态结果，不影响基础功能体验

#### 2. Web 前端 `.env` 配置（推荐）

在 `web` 目录下创建 `.env` 文件，最简单的配置如下：

```env
REACT_APP_API_URL=http://localhost:3000/api
REACT_APP_UPLOAD_URL=http://localhost:3000/uploads
```

说明：

- `REACT_APP_API_URL`：前端请求后端 API 的根地址
- `REACT_APP_UPLOAD_URL`：上传文件访问地址

前端里凡是以 `REACT_APP_` 开头的变量，会在构建时自动注入到代码中。

### 五、初始化 MySQL 数据库（超详细）

> 目标：在 MySQL 中创建本项目需要的库和表，并填充一些基础数据/测试用户。

#### 方式一：用项目提供的 Node.js 脚本（最推荐）

1. 打开 PowerShell，进入后端目录：

```bash
cd c:\Users\12556\Desktop\Demo\ihc\backend
```

2. 确保 `.env` 中的数据库配置与你本机 MySQL 一致（特别是用户名、密码）

3. 运行初始化脚本：

```bash
npm run db:init
```

该命令会执行 `database/init.sql`，在 MySQL 中创建数据库 `ihc` 和所需表结构，并插入部分测试数据（如测试用户）。

执行成功后，你的数据库里会出现这些表（部分示例）：

- `users` 用户表
- `artworks` 作品表
- `products` 产品表
- `orders` 订单表
- `ar_contents` AR 内容表

#### 方式二：使用 MySQL 命令行（了解原理用）

1. 打开 PowerShell，进入项目根目录：

```bash
cd c:\Users\12556\Desktop\Demo\ihc
```

2. 登录 MySQL：

```bash
mysql -u root -p123456
```

3. 在 MySQL 命令行中执行脚本（注意路径）：

```sql
source backend/database/init.sql;
```

或在系统命令行中直接执行：

```bash
mysql -u root -p123456 < backend/database/init.sql
```

> 如果你的 root 密码不是 `123456`，请改成你自己的。

#### 方式三：MySQL Workbench（图形界面）

1. 打开 MySQL Workbench
2. 新建连接，填入你安装 MySQL 时设定的 root 账号和密码
3. 打开 `backend/database/init.sql` 文件
4. 点击执行按钮（闪电图标），执行整个脚本

#### 验证数据库是否初始化成功

无论使用哪种方式，初始化完成后，在 MySQL 中执行：

```sql
SHOW DATABASES;
USE ihc;
SHOW TABLES;
```

能看到多个与业务相关的表（如 `users`、`artworks` 等），说明数据库准备就绪。

> **遇到报错怎么办？**
> - 报错「Access denied」：检查用户名/密码是否正确
> - 报错「Can’t connect to MySQL server」：检查 MySQL 服务是否启动（在「服务」里找到 MySQL，保证状态为“正在运行”）

### 六、启动后端服务（Node.js + Express）

后端代码在 `backend` 目录，入口文件是 `src/index.js`，通过 `nodemon` 以开发模式运行。

1. 打开一个新的 PowerShell 窗口
2. 进入后端目录：

```bash
cd c:\Users\12556\Desktop\Demo\ihc\backend
```

3. 启动开发模式：

```bash
npm run dev
```

如果一切正常，你会在终端看到类似日志：

- ✅ MySQL 数据库连接成功
- 服务启动成功，监听 `http://localhost:3000`

#### 常见启动问题排查（后端）

- **端口被占用（EADDRINUSE: port 3000）**
  - 说明本地已经有程序占用了 3000 端口
  - 解决方案：
    - 把 `.env` 里的 `PORT` 改成其它端口（例如 3001，再重启后端）
    - 或在任务管理器中结束占用 3000 端口的进程
- **无法连接数据库**
  - 查看终端打印的错误信息中是否有「MySQL」相关提示
  - 检查：
    - MySQL 是否已启动
    - `.env` 中 `DB_HOST/DB_PORT/DB_USER/DB_PASSWORD/DB_NAME` 是否正确
    - 防火墙/安全软件是否拦截了本地 MySQL

### 七、启动 Web 前端（React）

1. 再打开一个新的 PowerShell 窗口（保持后端窗口不要关）
2. 进入前端目录：

```bash
cd c:\Users\12556\Desktop\Demo\ihc\web
```

3. 启动前端开发服务器：

```bash
npm start
```

首次启动会进行打包编译，时间可能稍长一点。成功后，终端一般会提示类似信息：

- `Starting the development server...`
- `You can now view ihc-web in the browser.`
- `Local: http://localhost:3000`

浏览器会自动打开（或手动在浏览器地址栏输入）：

```text
http://localhost:3000
```

你应该可以看到项目的 Web 界面。如果页面能正常展示、操作不报错，说明前端启动成功。

#### 前端端口与后端的关系

- 在 `web/package.json` 中配置了 `"proxy": "http://localhost:3000"`  
- 或通过 `.env` 中的 `REACT_APP_API_URL=http://localhost:3000/api` 指定后端地址

当你在前端代码中使用 `axios` 或 `fetch` 请求 `/api/...` 时，请求会被转发到后端服务。

> **注意**：如果你把后端端口改成了 3001，记得同步修改：
> - `backend/.env` 中 `PORT=3001`
> - `web/.env` 中 `REACT_APP_API_URL=http://localhost:3001/api`
> - 如有必要，移除 `web/package.json` 中原来的 `"proxy"` 或改成对应端口

### 八、一套标准「从零到跑起来」流程复盘（直接照抄即可）

如果你完全没改配置，且使用默认端口和数据库账号，可以按这一套命令顺序来操作：

1. 打开 PowerShell，进入项目根目录：

```bash
cd ...\ihc
```

2. 安装依赖（只需第一次执行）：

```bash
npm run install:all
```

3. 配置环境变量：

- 在 `backend` 里创建 `.env`，填入文中示例（保证 DB 和端口设置正确）
- 在 `web` 里创建 `.env`，配置 `REACT_APP_API_URL` 和 `REACT_APP_UPLOAD_URL`

4. 初始化数据库：

```bash
cd backend
npm run db:init
cd ..
```

5. 启动后端（保持窗口打开）：

```bash
cd backend
npm run dev
```

6. 新开一个 PowerShell 窗口，启动前端：

```bash
cd ...\ihc\web
npm start
```

7. 在浏览器访问：

```text
http://localhost:3000
```

看到页面、能正常注册/登录/浏览内容，就表示前后端 + 数据库整体环境搭建成功。
