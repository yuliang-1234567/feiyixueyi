# 非遗平台（IHC）阿里云部署指南

本文档说明如何将本项目部署到阿里云 ECS，包括服务器选型、环境准备与部署步骤。

---

## 一、项目技术栈概览

| 组件   | 技术           | 说明                    |
|--------|----------------|-------------------------|
| 后端   | Node.js + Express | 端口默认 3000           |
| 前端   | React (CRA)    | 构建后为静态文件        |
| 数据库 | MySQL 5.7/8.0  | Sequelize ORM           |
| 依赖   | 火山引擎/DeepSeek/OpenAI 等 API | 需配置密钥 |

---

## 二、需要购买的阿里云产品

### 1. 云服务器 ECS（必选）

- **推荐规格**（按访问量选择）：
  - **入门/演示**：1 核 2GB，40GB 系统盘，按量付费或包月约 ￥70–100/月
  - **小规模正式**：2 核 4GB，60GB 系统盘，约 ￥150–250/月
  - **有高并发**：4 核 8GB 及以上
- **镜像**：**Alibaba Cloud Linux 3** 或 **Ubuntu 22.04**
- **地域**：选离用户近的（如华东、华北）
- **安全组**：需放行 80、443、22、3000（若不用 Nginx 反向代理则必须放行 3000）

购买入口：阿里云控制台 → 云服务器 ECS → 创建实例。

### 2. 云数据库 RDS MySQL（可选，推荐生产环境使用）

- 可用 ECS 自建 MySQL，也可使用 **RDS MySQL** 更省心、有备份与高可用。
- 若自建：与 ECS 同地域、同 VPC 更佳；若用 RDS，创建时选与 ECS 同 VPC 或内网互通。

### 3. 域名（可选）

- 若要用域名访问：在阿里云「域名」购买并备案（大陆服务器必须备案）。
- 不备案可先用 **ECS 公网 IP** 访问。

### 4. SSL 证书（可选）

- 若需 HTTPS：在「数字证书管理服务」申请免费证书（如 DV 单域名），或使用 Let’s Encrypt。

## 三、部署前准备（本地/控制台）

### 1. 环境变量清单

在服务器上需要配置以下变量（建议用 `backend/.env`，不要提交到 Git）。

**后端 `backend/.env`：**

```env
# 运行环境
NODE_ENV=production
PORT=3000

# 数据库（与服务器上 MySQL 一致）
DB_HOST=localhost
DB_PORT=3306
DB_NAME=ihc
DB_USER=你的数据库用户名
DB_PASSWORD=你的数据库密码

# JWT（必填，生产环境务必用强随机字符串）
JWT_SECRET=请替换为至少32位随机字符串
JWT_EXPIRE=7d

# 跨域（改成你的前端访问地址，如 https://你的域名.com）
CORS_ORIGIN=https://你的域名.com

# 以下按需配置，不用的可留空或删除
# DeepSeek
DEEPSEEK_API_KEY=

# 火山引擎 即梦
VOLCENGINE_ACCESS_KEY=
VOLCENGINE_SECRET_KEY=
VOLCENGINE_REGION=cn-north-1

# OpenAI（若使用）
OPENAI_API_KEY=
```

**前端构建时的 API 地址：**

在构建前端前，设置环境变量（或在 CI/部署脚本里 export）：

```bash
# Linux/Mac 在构建前执行
export REACT_APP_API_URL=https://你的域名.com/api
# 若暂时用 IP： export REACT_APP_API_URL=http://你的ECS公网IP:3000/api
```

然后执行 `npm run build`（见下文）。

### 2. 代码与构建

- 确保本机可正常 `npm run build:web`、后端可 `npm start` 并连上数据库。
- 将代码上传到服务器：Git 克隆（推荐）或本地上传压缩包再解压。

### 3. 服务器侧准备

- 已购买 ECS，并获取 **公网 IP**、**root 或 sudo 账号密码/密钥**。
- 安全组已放行：**22（SSH）、80（HTTP）、443（HTTPS）、3000（后端，若直接暴露）**。

## 四、服务器环境安装（以 Ubuntu 22.04 为例）

以下命令均在 ECS 上执行（SSH 登录后）。

### 1. 更新系统并安装基础工具

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git
```

### 2. 安装 Node.js 18+

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
node -v   # 应 >= 18
npm -v
```

### 3. 安装 MySQL

```bash
sudo apt install -y mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql
sudo mysql_secure_installation   # 按提示设置 root 密码、禁用远程 root 等
```

创建项目数据库与用户：

```bash
sudo mysql -u root -p
```

在 MySQL 中执行：

```sql
CREATE DATABASE ihc CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'ihc_user'@'localhost' IDENTIFIED BY '你的强密码';
GRANT ALL PRIVILEGES ON ihc.* TO 'ihc_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

将上面「你的强密码」与 `backend/.env` 中 `DB_USER`、`DB_PASSWORD` 保持一致。

### 4. 安装 Nginx（做反向代理与静态资源）

```bash
sudo apt install -y nginx
```

### 5. 安装 PM2（进程守护）

```bash
sudo npm install -g pm2
```

## 五、部署步骤

### 1. 上传/拉取代码到服务器

例如放到 `/var/www/ihc`：

```bash
sudo mkdir -p /var/www/ihc
sudo chown $USER:$USER /var/www/ihc
cd /var/www/ihc
git clone 你的仓库地址 .
# 若无 Git：用 scp/sftp 上传压缩包后解压到 /var/www/ihc
```

### 2. 安装依赖并构建前端

```bash
cd /var/www/ihc

# 后端依赖
npm run install:all
# 或分别：cd backend && npm install && cd ../web && npm install

# 设置前端 API 地址并构建（替换为你的域名或 IP）
export REACT_APP_API_URL=https://你的域名.com/api
cd web && npm run build && cd ..
```

构建产物在 `web/build`。

### 3. 配置后端环境变量

```bash
cp backend/.env.example backend/.env 2>/dev/null || true
nano backend/.env   # 按「三、1」填写并保存
```

务必设置 `NODE_ENV=production`、`JWT_SECRET`、数据库与 `CORS_ORIGIN`。

### 4. 初始化数据库（若尚未执行过）

```bash
cd /var/www/ihc/backend
node database/execute-sql.js    # 若有初始化 SQL
# 或
node database/execute_migration.js
```

按你项目实际使用的脚本执行一次即可。

### 5. 用 PM2 启动后端

```bash
cd /var/www/ihc/backend
pm2 start src/index.js --name ihc-api
pm2 save
pm2 startup   # 按提示执行生成的命令，实现开机自启
```

检查：

```bash
pm2 status
curl http://localhost:3000/api/health
```

### 6. 配置 Nginx

创建站点配置（将 `你的域名.com` 和 `你的ECS公网IP` 替换为实际值）：

```bash
sudo nano /etc/nginx/sites-available/ihc
```

写入（**HTTPS 可选，先用 HTTP 可先注释 ssl 相关行**）：

```nginx
server {
    listen 80;
    server_name 你的域名.com 你的ECS公网IP;
    # 若已配置 HTTPS，可在此 server 内重定向到 443

    root /var/www/ihc/web/build;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    location /uploads {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
    }
}
```

启用并测试：

```bash
sudo ln -s /etc/nginx/sites-available/ihc /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 7. 安全组与防火墙

- **阿里云安全组**：放行 80、443、22；若 Nginx 已反向代理到 3000，可不放行 3000。
- **本机防火墙**（若开启）：
  ```bash
  sudo ufw allow 80
  sudo ufw allow 443
  sudo ufw allow 22
  sudo ufw enable
  ```

## 六、部署后验证

1. 浏览器访问：`http://你的域名或公网IP`，应打开前端页面。
2. 登录、调用接口、上传等走 `/api`，应都由 Nginx 转发到后端 3000 端口。
3. 查看后端日志：`pm2 logs ihc-api`。
4. 若改代码或环境变量：  
   - 改后端：`pm2 restart ihc-api`  
   - 改前端：在 `web` 目录重新 `export REACT_APP_API_URL=... && npm run build`，无需重启 PM2。

## 八、常见问题

- **502 Bad Gateway**：多半是后端未启动或端口不对，检查 `pm2 status` 和 `curl http://127.0.0.1:3000/api/health`。
- **接口 404**：确认 Nginx 中 `location /api` 的 `proxy_pass` 为 `http://127.0.0.1:3000`，且带 `/api` 的请求会转发到后端。
- **跨域错误**：确认 `CORS_ORIGIN` 与前端访问的域名一致（含协议），且 Nginx 已正确转发。
- **数据库连接失败**：检查 `DB_HOST`（本机 MySQL 用 `localhost`）、用户名密码、MySQL 是否允许本地连接。

按上述步骤即可在阿里云 ECS 上完成部署。若你提供当前使用的镜像（如 Ubuntu 版本）和是否有域名，可以再写一版更贴你环境的「最小步骤版」或自动化脚本。
