# 非遗AI平台项目答辩总结

## 一、网站名字

- **全称**：AI驱动的“非遗”技艺沉浸式学习与文创平台  
- **简称**：非遗AI学艺与文创平台（IHC）

---

## 二、技术实现与使用工具

### 1. 总体架构

- 采用前后端分离架构：`Web前端 + 后端API + MySQL数据库`
- 支持从学习、创作到交易的完整业务链路

### 2. 前端技术

- `React`：构建交互式单页应用
- `Ant Design`：快速搭建统一风格的UI组件
- `ECharts`：实现中国地域非遗地图可视化与交互
- `Axios`：统一管理接口请求

### 3. 后端技术

- `Node.js + Express`：提供RESTful API服务
- `Sequelize + MySQL`：实现数据建模与持久化
- `JWT + bcryptjs`：完成身份认证与密码安全
- `Multer`：处理用户上传图片与文件校验

### 4. AI与图像能力

- `Sharp`：图像缩放、灰度化、归一化与像素处理
- `DeepSeek API`：生成“非遗导师式”个性化学习建议

---

## 三、算法部分（非遗AI训练与优化，让AI更“懂行”）

### 1. AI学艺评分算法

- 将用户作品与参考作品统一到 `224×224`
- 进行灰度化与归一化后，计算像素级差异
- 通过指数映射得到相似度，再转换为百分制评分

核心思路：

1. 提取两张图像的原始像素数据  
2. 计算逐像素差异并求平均差异  
3. 归一化差异后用指数函数映射为相似度  
4. 输出评分并联动建议模块

### 2. AI建议生成机制

- **规则层**：按相似度区间输出针对性改进建议
- **模板层**：内置多套“优势-不足-学习路径”建议模板
- **大模型层**：调用 DeepSeek 输出结构化 JSON 建议（学习方向、专业提升、技法重点、导师资源）

### 3. 数字焕新优化策略

- 输入支持：纹样图、文本描述、图文组合
- 自动分流：图生图/文生图的最优调用路径
- 参数优化：按产品类型配置融合参数（位置、缩放、透明度、遮罩形状等）

### 4. 后续可强化方向（让AI更“懂行”）

- 建立非遗知识库与 RAG 检索增强
- 引入专家标注数据，做更细粒度的监督优化
- 从“单一相似度”升级到“构图+线条+风格+文化语义”多维评价
- 构建用户学习画像，实现个性化训练路径推荐
